🗂️ From Raw Data to P&L — The Workflow

In real companies, financial data doesn't start as a neat P&L. It starts as a log of individual transactions — invoices, payments, fees — stored in a Raw Data sheet.

Your job as a business analyst is to aggregate that raw log into meaningful P&L lines. The tool for that: SUMIF.

Raw Data sheet → 60 rows of transactions
→ SUMIF by category
→ P&L sheet → 10 summary lines
→ Pivot Table → visual cross-check
➕

SUM / AVERAGE

=SUM(range) to total a column. =AVERAGE(range) for the mean. Used in P&L totals and for average order value or average unit cost.

✖️

Multiply & Divide

=B2*C2 for Qty × Price. =B5/B2 for Margin %. Already familiar — today these appear as helper formulas inside larger expressions.

🔀

IF / IFS

=IF(condition, value_if_true, value_if_false) — conditional logic. Use to flag whether a business is profitable, or classify margin levels.

🔍

SUMIF

=SUMIF(range, criteria, sum_range) — sum only the rows that match a condition. Core formula for building a P&L from raw transactions.

🛡️

IFERROR

=IFERROR(formula, fallback) — wraps any formula and catches errors like #DIV/0! or #N/A. Keeps your model clean when data is missing.

🏷️

Cell Naming

Click a cell → click the Name Box (top-left, shows the address) → type a name → Enter. Now use that name in any formula: "=Salaries_Mo*12" instead of "='Fixed Costs'!B5*12". Readable and error-resistant.

🎨

Cell Formatting

Ctrl+Shift+$ = Currency (€). Ctrl+Shift+% = Percentage. Ctrl+1 for full Format Cells dialog. Freeze top row: View → Freeze Panes.

📖

VLOOKUP

=VLOOKUP(value, table, col, FALSE) — look up a value in a reference table and return a matching result. Used to cross-reference rate cards and price lists.

📊

Pivot Table

Insert → PivotTable. Summarises and cross-tabulates raw data in seconds without any formulas. The fastest way to verify your SUMIF results.

🔍 SUMIF — The Formula That Builds Your P&L

SUMIF scans a column for a matching value and sums only the corresponding amounts. Think of it as a filter + sum in one step.

# Syntax
=SUMIF(range, criteria, sum_range)

# Example: sum all Brand Animation revenue from Raw Data
=SUMIF('Raw Data'!D:D, "Brand Animation", 'Raw Data'!H:H)

# Reading it aloud:
"In the Category column (D), find every row that says 'Brand Animation'
and add up the corresponding Total (H) values."

# SUMIFS — two conditions at once (Revenue only)
=SUMIFS('Raw Data'!H:H, 'Raw Data'!C:C, "Revenue", 'Raw Data'!D:D, "Brand Animation")
💡
The category name in your formula must exactly match the text in the Raw Data column — but capitalisation doesn't matter. "Brand Animation" = "brand animation" = "BRAND ANIMATION".

📖 VLOOKUP — Looking Up Rates from a Reference Table

Your workbook has a Reference sheet with standard unit rates. VLOOKUP lets you pull those rates into your P&L automatically — no copy-pasting.

# Syntax
=VLOOKUP(lookup_value, table_array, col_index, FALSE)

# Example: look up the standard rate for "Brand Animation"
=VLOOKUP("Brand Animation", Reference!A:C, 2, FALSE)

# Arguments explained:
"Brand Animation" ← what to search for (lookup value)
Reference!A:C ← where to search (the reference table)
2 ← return column 2 (Standard Rate) from the table
FALSE ← exact match only (always use FALSE)

# XLOOKUP — the modern replacement (Excel 2019+)
=XLOOKUP("Brand Animation", Reference!A:A, Reference!B:B)
⚠️
Always use "FALSE" as the last argument in VLOOKUP. Without it, Excel looks for an approximate match and can return wrong results — a common source of silent errors in financial models.

🔀 IF / IFS — Flagging Business Outcomes

Use IF to add logic to your P&L: is the business profitable? Is the margin healthy? These flags turn a spreadsheet into a decision tool.

# IF — one condition, two outcomes
=IF(C27 > 0, "✅ Profitable", "❌ Loss")

# IFS — multiple conditions (Excel 2019+)
=IFS(C15/C8 > 0.5, "High margin",
     C15/C8 > 0.3, "Moderate",
     TRUE, "Low margin")

# Nested IF — if you don't have IFS available
=IF(C15/C8 > 0.5, "High", IF(C15/C8 > 0.3, "Moderate", "Low"))

📊 Pivot Tables — Your Cross-Check Tool

Once your P&L is built, insert a Pivot Table from the Raw Data to verify every number independently — without writing a single formula.

  1. Click any cell inside your Raw Data sheet.
  2. Insert → PivotTable → place it in a new sheet.
  3. Drag Category to the Rows area.
  4. Drag Total (€) to the Values area (it sums automatically).
  5. Drag Type to the Filters area → filter to "Revenue" only, then "Cost" only.
  6. Compare the Pivot totals against your P&L cells. They should match.
💡
If a Pivot total differs from your SUMIF, the category name in one of your Raw Data rows likely has a typo or extra space. Pivot Tables make this kind of error immediately visible.

📌 Quick Reminder: Relative vs. Absolute References

When you copy a SUMIF formula down the column, the criteria reference (e.g. the category cell in column A) should update row by row — but the Raw Data range should stay fixed.

# Use $ to lock the Raw Data range so it doesn't shift when you copy down
=SUMIF('Raw Data'!$D:$D, A5, 'Raw Data'!$H:$H)

# A5 is relative — it changes to A6, A7… as you drag down
# $D:$D and $H:$H are absolute — they always point to the same columns

# Press F4 after clicking a cell reference to add/remove $ signs
⚡
Manual First, AI Second. Try to write every formula yourself before asking for help. When you're stuck, give Gemini full context: "I'm writing a SUMIF formula in Excel. My Raw Data is on a sheet called 'Raw Data'. Category names are in column D, transaction totals are in column H. I want to sum all rows where category = 'Brand Animation'. What's the formula?"

Nomad Bean ☕ has 16 revenue transactions spread across the year — four billing batches for each of their four subscription plans. Here's how SUMIF collapses those 16 rows into 4 clean P&L lines.

Raw Data sheet — 16 rows (sample)
TxID Type Category Qty Total (€)
INV-001 Revenue Explorer Subscription 45 674.55
INV-002 Revenue Connoisseur Subscription 40 919.60
INV-003 Revenue Roasters Choice 20 699.80
INV-004 Revenue B2B Corporate Orders 1 850.00
… 12 more rows across Q2–Q4 …
✅ P&L sheet — SUMIF formulas
Line Item Formula (you write this) Result
Explorer Subscription =SUMIF('Raw Data'!D:D,"Explorer Subscription",'Raw Data'!H:H) 2,698.20
Connoisseur Subscription =SUMIF('Raw Data'!D:D,"Connoisseur Subscription",'Raw Data'!H:H) 3,678.40
Roasters Choice =SUMIF('Raw Data'!D:D,"Roasters Choice",'Raw Data'!H:H) 2,799.20
B2B Corporate Orders =SUMIF('Raw Data'!D:D,"B2B Corporate Orders",'Raw Data'!H:H) 10,200.00

One formula per line. Each one scans all 60+ rows in Raw Data and pulls only the matching transactions. Update any raw transaction — the P&L updates instantly.

VLOOKUP CROSS-CHECK
← Reference sheet has: "Explorer Subscription" | 14.99 | "per box / month"
=VLOOKUP("Explorer Subscription", Reference!A:C, 2, FALSE)
→ Returns: 14.99

← Divide SUMIF result by standard rate to verify row count
= 2698.20 / 14.99 = 180 boxes  ✅ matches Raw Data (45 × 4 quarters)

Hands-On: Build Your P&L from Raw Data

~90 minutes · Your company's financial model has 4 sheets — work through them in order

📁
Your File: "[company_slug]_financial.xlsx" Your workbook has five sheets: Instructions → Raw Data → Reference → Fixed Costs → P&L. Raw Data contains all FY 2025 transactions. Fixed Costs holds the monthly unit values you'll name and reference. Your job is to fill every yellow cell in the P&L with the correct formula — each cell has a plain-English hint telling you what the formula should do. Read the Instructions sheet first.

Part 1 — Explore the Data (10 min)

  1. Read the Instructions sheet top to bottom. Note the colour legend: 🟦 Blue = given, 🟨 Yellow = write formula, 🟩 Green = auto-calculated.
  2. Open the Raw Data sheet. Scroll through it. Notice: each row is one transaction with a Date, TxID, Type (Revenue/Cost), Category, Description, Qty, Unit Price, and Total.
  3. Open the Reference sheet. This is your rate card — standard unit prices for every category. You will use this for VLOOKUP.
  4. Count how many distinct revenue categories your company has. These become the revenue lines in your P&L.

Part 2 — Build Revenue & COGS with SUMIF (30 min)

  1. Go to the P&L sheet. Read the formula hint in column B next to the first Revenue line — it tells you exactly which SUMIF to write.
  2. Click the yellow cell in the "Your Formula" column (column C) next to the first revenue line. Type the SUMIF formula using the category name shown in column A. Example:
    "=SUMIF('Raw Data'!D:D,"Brand Animation",'Raw Data'!H:H)"
  3. Repeat for every revenue line. Each line has a different category name.
  4. The TOTAL REVENUE cell is already written (green). Verify it auto-sums your revenue lines correctly.
  5. Repeat the same SUMIF process for every COGS line (Section 2). Cost transactions also live in Raw Data — SUMIF finds them by category.

Part 3 — VLOOKUP, Gross Profit & Cell Naming (30 min)

  1. Fill in the Gross Profit cell (Section 3) — the hint says "Subtract Total COGS from Total Revenue". Write that as an Excel formula referencing the two total cells above.
  2. Fill the Gross Margin % cell — divide Gross Profit by Total Revenue. Then format the cell as a percentage (Ctrl+Shift+%).
  3. Find the VLOOKUP Check section. Read the hint for each cell — it describes what to look up and where. Write both VLOOKUP formulas using the Reference sheet.
  4. Now go to the Fixed Costs sheet. Read the instructions at the top. For each monthly cost cell, follow the 4-step naming guide to assign the suggested cell name (e.g. "Salaries_Mo").
  5. Return to the P&L sheet, Section 4. The hint for each fixed cost says which named cell to reference — write the formula as "=CellName * 12". After naming the cell on the Fixed Costs sheet, Excel will recognise the name anywhere in the workbook.
  6. Section 5 — EBIT: the hint says "Subtract Total Fixed Costs from Gross Profit". Write it.

Part 4 — IF Flags & Pivot Table (30 min)

  1. Section 6 — Business Status (IF): write "=IF(EBIT_cell > 0, "✅ Profitable", "❌ Loss")".
  2. Next cell — Margin Status (IFS): write an IFS formula that returns "High margin" if gross margin > 50%, "Moderate" if > 30%, and "Low margin" otherwise.
  3. Now go back to the Raw Data sheet. Click any cell inside the data. Go to Insert → PivotTable → place it in a new sheet called "Pivot Check".
  4. In the Pivot: Rows = Category, Values = Sum of Total, Filters = Type. Filter to "Revenue" — do the category totals match your P&L revenue lines? Then filter to "Cost" — do your COGS values match?
  5. If any number is off by more than €1, trace back: check the category name spelling in Raw Data and in your SUMIF formula.
✅
Quality Check — your model is correct when: (1) every yellow cell has a formula, not a typed number; (2) the Pivot Table totals match your P&L; (3) the Business Status cell shows the right flag based on your EBIT. If the Pivot differs from your SUMIF — check for a typo in the category name.
